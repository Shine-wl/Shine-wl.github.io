git 基本使用

2017-09-20
git

操作计算机的两种方式

  1. 图形界面–将命令转换为图形
  2. 命令行 –命令窗口

    命令的通用规则

  • 命令 [选项] [参数]–后面两个可选
    • ls -al —-命令-选项
    • ls -a ./abc –命令-选项-参数

      DOS 和 shell 的区别

  • 工作环境
    • dos(window)
    • shell(linus)–分类：csh ksh bash

      shell bash 常用命令

  • pwd(print Working Directory)查看当前目录
  • cd(change Directory)切换目录，如cd/hello
  • ls(List)查看当前目录下的内容

  1. ls 查看的是当前目录下的文件，或文件夹

  2. ls -a 查看的还包括./和../

  3. ls -a -l可以简写ls -al查看的是详细内容，包括文件类型创建日期等等…

  • mkdir(Make Directory)创建目录，如mkdir hello
  • touch创建文件,如touch index.html
  • cat查看文件全部内容，如cat index.html
  • less查看文件，如less index.html
  • rm(remove)删除文件，如rm index.html
  • rm hello -rf删除文件夹，但是要谨慎使用，删除后不进回收站，不容易找回
  • rmdir(Remove Directory)删除文件，但是只能删除空文件
  • mv(move)移动文件或重命名，如mv index.html ./demo/index.html
  • cp(copy)复制文件，cp index.html ./demo/index.html
  • tab自动补全，连续按两次，会将匹配到的内容显示出来

  • 和>>重定向，如echo hello world >index.txt,>覆盖 >>追加

  • |管道符可以将多个命令连接使用，上一个命令的执行结果当做下一个命令的参数
  • grep匹配内容，一般结合管道符使用

    vi编辑器

    linux操作系统标配，相当于windows下的记事本

    vi编辑器的三种模式

    vi编辑器提供了3种模式，分别是命令模式、插入模式、底行模式，每种模式下用户所能进行的操作是不一样的。

  • 三种模式切换如下图所示

    vi常用操作

    底行模式

  • 打开/创建文件，vi文件路径
  • 底行模式:w保存,:w 文件名另存为
  • 底行模式:wq保存并退出
  • 底行模式:e! 撤销更改，返回到上一次保存的状态
  • 底行模式:q! 不保存强制退出
  • 底行模式:set nu 设置行号

    命令模式

  • 命令模式u辙销操作，可多次使用
  • 命令模式dd删除当前行
  • 命令模式yy复制当前行
  • 命令模式p粘贴内容
  • 命令模式ctrl+f向前翻页
  • 命令模式ctrl+b向后翻页
  • 命令模式i进入编辑模式，当前光标处插入
  • 命令模式I进入编辑模式，光标移动到行首
  • 命令模式a进入编辑模式，当前光标后插入
  • 命令模式A进入编辑模式，光标移动到行尾
  • 命令模式o进入编辑模式，当前行下面插入新行
  • 命令模式O进入编辑模式，当前行上面插入新行

    git的相关概述

  1. 文件的状态
    1. 未跟踪 (位于工作目录，新创建的文件)
    2. 以暂存 (位于暂存区的文件，一般用于存储半成品)
    3. 已提交 (位于本地仓库的文件，一般用于存储成品，该成品最终会上线)
    4. 已修改 (位于工作目录，暂存过的文件又被修改了)
  2. git相关的3个工作区域
    1. 工作目录（.git所在的目录，用来存储当前代码）
    2. 暂存区（git临时存储代码，.git文件夹里）
    3. 本地仓库（存储整个项目的快照，保存当前的暂存区中的一份代码，.git文件夹里）
