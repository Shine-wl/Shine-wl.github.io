this 指向问题

2016-07-10
this

具体执行谁取决于函数的调用方式

  • 普通函数调用是 window， 严格模式下是 undefined

  • 构造函数的实例对象调用是 实例对象， 原型方法中 this 也是实例对象

  • 对象方法调用 该方法所属对象 紧挨着的对象

  • 事件绑定 绑定事件的对象

    • 这里吧person作为 btn.click 的事件处理函数
    • 事件处理函数不是用户调用的，而是系统来调用
    • 事件处理函数内部的 this 指向DOM对象

  • 定时器 window

    • 这里是将 Person 作为定时器的处理函数
    • 当事件到达，系统会帮你调用Person
    • 事件处理函数内部的this指向window

函数内this指向不受外部环境影响

数组遍历内的this指向问题

  • 对于数组的一些遍历方法：
    • 例如 forEach、map、filter、some 等，内部的 this 都指向 window
    • 无论这个代码的执行处于什么环境都是 window，也不受外部影

call方法内的this指向

  • 函数的call方法可以实现改变this指向 然后调用该函数
